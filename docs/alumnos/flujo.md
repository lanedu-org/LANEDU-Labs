# Flujo del alumno

El flujo de trabajo en LANEDU Labs es simple, repetible y siempre el mismo para todos los Labs.

Todo ocurre en GitHub.

---

## Paso 1 — Elegir un Lab

Desde la plataforma, selecciona un Lab disponible dentro de una Ruta.

Lee cuidadosamente:
- La historia
- El objetivo
- Las reglas del Lab

Asegúrate de entender **qué se espera**, no cómo hacerlo.

---

## Paso 2 — Fork del repositorio

Haz un fork del repositorio del Lab a tu cuenta de GitHub.

El repositorio base no se modifica.  
Todo tu trabajo ocurre en tu fork.

---

## Paso 3 — Trabajar en la solución

Implementa la solución:

- Usa GitHub Codespaces o tu entorno local
- Realiza commits con tu progreso
- Documenta decisiones si lo consideras necesario

Puedes investigar, probar y corregir libremente.

---

## Paso 4 — Crear el Pull Request

Cuando tu solución esté lista:

- Abre un Pull Request desde tu fork
- Apunta a la rama `main` del repositorio base
- Usa el formato de título definido por el Lab

El Pull Request es tu entrega.

---

## Paso 5 — Verificar progreso

Vuelve a la plataforma y presiona **Verificar progreso**.

LANEDU Labs:
- Detecta tu Pull Request
- Aplica las reglas del Lab
- Actualiza el estado automáticamente

---

## Paso 6 — Resultado

El Lab puede quedar en uno de estos estados:
- Pendiente
- PR detectado
- Completado

Si algo no cumple las reglas, podrás corregirlo actualizando el mismo Pull Request.

---

## Importante

- No hagas merge del Pull Request
- No abras múltiples PR para el mismo Lab
- No modifiques archivos prohibidos

Sigue siempre las reglas definidas en el Lab.

---

## En resumen

El flujo del alumno es:

1. Elegir Lab  
2. Fork  
3. Resolver  
4. Pull Request  
5. Validación automática  

Siempre igual. Para todos los Labs.
