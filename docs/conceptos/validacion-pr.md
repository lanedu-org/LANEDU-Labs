# Validación por Pull Requests

LANEDU Labs valida los desafíos usando **Pull Requests en GitHub**, del mismo modo en que se valida el trabajo en equipos reales.

No existen formularios, evaluaciones manuales ni entregas fuera de GitHub.

---

## El Pull Request como entrega

En LANEDU Labs, un Pull Request representa la entrega oficial de un Lab.

El Pull Request contiene:
- El trabajo realizado
- El historial de commits
- Las decisiones técnicas
- La evidencia del proceso

Si el Pull Request cumple las reglas del Lab, el desafío se considera completado.

---

## Validación automática

La plataforma valida automáticamente los Labs leyendo información pública de GitHub.

Se verifican aspectos como:
- Existencia de un Pull Request abierto
- Título del Pull Request
- Rama de origen
- Archivos modificados
- Reglas específicas definidas por el Lab

No hay revisión manual por parte del equipo.

---

## Reglas claras y objetivas

Cada Lab define sus reglas de validación.

Estas reglas permiten que:
- Todos los usuarios sean evaluados bajo los mismos criterios
- La validación sea reproducible
- No exista ambigüedad en la entrega

El resultado es objetivo y transparente.

---

## Pull Requests que no se mergean

Los Pull Requests en LANEDU Labs **no se mergean**.

- El repositorio base permanece intacto
- Cada usuario trabaja en su fork
- El Pull Request existe solo como evidencia

Esto permite que múltiples personas trabajen en el mismo Lab sin conflictos.

---

## Evidencia pública y verificable

Toda la validación se basa en información pública:

- Repositorios
- Commits
- Pull Requests

Cualquier persona puede revisar el trabajo realizado.

Esto permite:
- Construir portafolio
- Demostrar habilidades reales
- Aprender revisando el trabajo de otros

---

## En resumen

La validación por Pull Requests permite:

- Aprender como se trabaja en equipos reales
- Evaluar con evidencia, no con teoría
- Escalar sin revisiones manuales
- Mantener transparencia total
