# ¿Qué es una Ruta?

Una **Ruta** es un conjunto de Labs organizados por un mismo dominio de conocimiento y ordenados por dificultad.

Las Rutas permiten avanzar de forma progresiva, desde problemas simples hasta desafíos más complejos, siempre a través de práctica real.

---

## Organización por dominio

Cada Ruta representa un área específica:

- Linux
- Git & GitHub
- Bases de Datos
- Docker
- Python
- Otros dominios técnicos

Una Ruta no mezcla temas.  
Se enfoca en desarrollar competencia práctica en un solo ámbito.

---

## Progresión de dificultad

Los Labs dentro de una Ruta están ordenados de:

**Fácil → Intermedio → Difícil**

Esto permite:
- Empezar sin experiencia previa
- Ganar confianza con problemas reales
- Enfrentar desafíos más complejos gradualmente

No es obligatorio completar todos los Labs, pero la progresión está pensada para construir habilidades de forma sólida.

---

## Independencia entre Rutas

Cada Ruta es independiente.

- Puedes comenzar por la Ruta que prefieras
- No necesitas completar una Ruta para iniciar otra
- Cada Ruta desarrolla habilidades distintas

Esto permite adaptar el aprendizaje a tus objetivos personales o profesionales.

---

## Evidencia por Ruta

Completar Labs dentro de una Ruta genera evidencia acumulativa:

- Historial de Pull Requests
- Proyectos relacionados
- Progresión visible en complejidad

Una Ruta completada demuestra dominio práctico de un área específica.

---

## Rutas, no cursos

Una Ruta no es un curso tradicional.

- No hay clases
- No hay lecciones obligatorias
- No hay exámenes

Solo una secuencia de problemas reales diseñados para aprender trabajando.

---

## En resumen

Una Ruta es:
- Un dominio técnico
- Una secuencia de Labs
- Una progresión real de dificultad
- Evidencia acumulada de aprendizaje práctico
